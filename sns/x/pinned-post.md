```
「成績、ちゃんと残したいけど続かない…」
```

## 設計の意図

参考にしたAny Plannerの固定ポスト構造を踏襲：

1. **共感フック**（"続かない…"でペインを言語化）
2. **アプリ定義**（誰のための・何アプリかを1文で）
3. **使い方の最小単位**（"打数や安打を入れるだけ"で導入障壁を下げる）
4. **機能箇条書き**（3点に絞り視認性を確保）
5. **リンク**（iOS / Web 両対応を明示）

## 添付画像の方針

固定ポストには **必ず画像を1枚添付** する。タイムラインでのサムネ視認性が固定ポストのCTRを大きく左右するため。

候補：
- ランキング画面のスクリーンショット（アプリのコア体験を1枚で見せる）
- 成績カード画像（自動計算された数値が並ぶ画面）
- アプリのモック3端末並列（既存のヘッダー画像と同系統）
