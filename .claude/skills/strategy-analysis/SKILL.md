# 戦略分析スキル

戦略系エージェント6体を並列起動し、データ分析・競合調査・施策立案・広告最適化を一括で実行する。

## 利用可能なエージェント

| エージェント | 役割 | model |
|---|---|---|
| `strategy-lead` | 全体戦略統括・ロードマップ・KPI・PDCA設計 | opus |
| `growth-marketing` | SEO施策・SNSマーケティング・ユーザー獲得/定着 | sonnet |
| `market-researcher` | 競合分析・市場調査・ポジショニング | sonnet |
| `product-planner` | 新機能/コンテンツ企画・コンバージョン導線設計 | sonnet |
| `ad-revenue-optimizer` | AdSense最適化・RPM改善・アフィリエイト戦略 | sonnet |
| `analytics-analyst` | GA/Search Consoleデータ分析・機会損失の特定 | sonnet |

## ワークフロー

### 1. データの準備

まず利用可能なデータを確認する:

```
data/google_analytics/ - GA4エクスポートデータ
data/google_ads/ - 広告収益データ
```

データファイルが存在する場合は全て読み込み、主要な指標をサマリーにまとめる:
- アクティブユーザー数、新規ユーザー数
- 流入元構成（organic, direct, SNS等）
- ランディングページ別のクリック数・表示回数・CTR・平均順位
- 検索クエリ別のクリック数・表示回数・CTR・平均順位
- ページ別PV・直帰率・エンゲージメント時間
- 広告収益データ

### 2. エージェントの並列起動

**6体全てをバックグラウンドで並列起動する。** 各エージェントにはステップ1で整理したデータサマリーを渡す。

各エージェントへのプロンプトには以下を含める:
- BUZZ BASEの概要（野球の個人成績をランキング形式で共有するWebアプリ）
- 現状のデータサマリー（ステップ1の結果）
- そのエージェントの専門領域に対応する分析依頼

#### strategy-lead への依頼内容
- 3ヶ月間の収益化ロードマップ（月別マイルストーン）
- KPI設計（追跡すべき指標と目標値）
- 施策の優先順位マトリクス（効果 x 工数）
- PDCAサイクルの設計
- 収益目標（3ヶ月後・6ヶ月後）
- 個人開発のリソース制約を考慮した現実的な計画

#### growth-marketing への依頼内容
- SEO施策（CTR改善、コンテンツ戦略、内部リンク戦略）
- SNSマーケティング施策（X/Instagram/TikTok）
- ユーザー獲得・定着施策
- 広告収益最大化のためのPV増加策
- 具体的なアクションプランと期待効果（優先度順）

#### market-researcher への依頼内容
- 「野球 計算ツール」「野球成績アプリ」市場の競合調査（Web検索で主要競合を特定）
- 競合と比較したBUZZ BASEの強み・弱み
- 検索需要から見える市場機会
- ターゲット（中高生野球部員）へのリーチ方法
- 類似スポーツアプリの成功事例と収益モデル

#### product-planner への依頼内容
- 新規コンテンツページの企画（解説ページ、新しい計算ツール）
- 既存ページの改善企画（CTR改善、直帰率低下、回遊率向上）
- SEO流入→アプリ登録のコンバージョン導線設計
- PVと広告収益を最大化するためのサイト構造提案
- GitHub Issueとして作成できるレベルの具体性

#### ad-revenue-optimizer への依頼内容
- AdSense設定の最適化（自動広告 vs 手動配置、推奨フォーマット）
- RPM改善の具体策（広告可視性、高単価コンテンツ、季節性活用）
- ページタイプ別の最適な広告配置位置
- AdSense以外の収益化オプション（アフィリエイト、スポンサー）
- 収益シミュレーション（PV x RPM別）

#### analytics-analyst への依頼内容
- トラフィックデータの詳細分析
- 機会損失の定量化（高表示・低CTRクエリの一覧と逸失クリック数）
- ユーザー行動パターンの分析（回遊、離脱ポイント）
- 前期比較（データがある場合）
- データに基づく施策の優先順位付け

### 3. 結果の統合

全エージェント完了後、以下の構成で統合レポートを作成する:

1. **全エージェント共通の認識** — 複数のエージェントが共通して最優先と判断した施策
2. **各エージェント固有の洞察** — それぞれの専門領域ならではの発見
3. **統合ロードマップ** — 全施策を優先度順に並べた実行計画
4. **KPIと収益目標** — 数値目標

### 4. ドキュメントの保存

統合レポートを `docs/strategy/` 配下に保存する。ファイル名は分析対象期間を含める:
- 例: `docs/strategy/seo-growth-plan-202604.md`
- 例: `docs/strategy/ad-revenue-optimization-202604.md`

## 注意事項

- 各エージェントは必ず `run_in_background: true` で起動し、並列実行する
- エージェントの完了を待つ間、ユーザーに進捗状況を伝える
- データが不足している場合は、利用可能なデータの範囲で分析する
- 全エージェントの結果はユーザーに分かりやすく統合して報告する
