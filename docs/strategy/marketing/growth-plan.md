### 競合環境

| サービス名 | 特徴 | BUZZ BASEとの差別化ポイント |
| ---------- | ---- | -------------------------- |
| PLAY by TeamHub | チーム単位のスコア管理・自動集計 | チーム管理ツール寄り。BUZZ BASEはチーム横断のランキング・SNS的共有に強み |
| ヒットメーカー | 個人打撃成績の記録・セイバーメトリクス分析（45,000DL） | 個人完結型。BUZZ BASEはグループ・ランキングによるソーシャル要素が差別化 |
| SCORE BASE | チーム管理・試合速報・スケジュール管理 | 管理ツール寄り。BUZZ BASEは「成績を見せ合う楽しさ」に特化 |
| .333 | 打撃成績の簡易記録 | 機能が限定的。BUZZ BASEはグループ・ランキング・野球ノートなど多機能 |

**BUZZ BASEの独自価値**: 「個人成績をランキング形式で友達と共有・比較できる」というソーシャル×競争要素は他アプリにない明確な差別化ポイント。

## 1. ユーザー獲得チャネル戦略

### ターゲットユーザーのSNS利用実態（2025年調査データ）

中高生のSNS利用率は97〜99%に達しており、主要SNSの毎日利用率は6〜8割と非常に高い。

| チャネル | 中高生利用率 | 特性 | BUZZ BASEとの相性 | 優先度 |
| -------- | ----------- | ---- | ----------------- | ------ |
| **Instagram** | 約70-89% | ビジュアル重視、ストーリーズ・リールが活発、情報発信意欲が最も高い | ランキング画像・成績カード共有と相性抜群 | **S（最優先）** |
| **TikTok** | 約53-61% | 短尺動画、バイラル力が最も高い、暇つぶし利用が中心 | 成績紹介・練習動画との組み合わせでバイラル狙い | **A（高）** |
| **X（旧Twitter）** | 約67-69% | リアルタイム性、情報拡散力、野球コミュニティが活発 | 大会速報・成績報告との連動、野球垢文化との親和性 | **A（高）** |
| **YouTube** | 約86% | 長尺コンテンツ、検索流入、ハウツー需要 | 使い方動画・成績分析解説で検索流入獲得 | **B（中）** |
| **LINE** | 約86-96% | インフラ的存在、1対1・グループコミュニケーション | チームメイトへの直接招待に活用 | **B（中・招待導線）** |

### チャネル別アクションプラン

#### Instagram（最優先）
- 公式アカウント運用開始（@buzzbase_jp）
- リール動画を週2-3本投稿（30秒以内の短尺）
- ストーリーズでユーザーの成績カードをリポスト
- プロフィールリンクにサービスURLを設置

#### TikTok（高優先）
- 公式アカウント開設
- 「成績を入力→ランキング表示」の操作デモ動画
- 野球あるある × BUZZ BASE の掛け合わせコンテンツ
- 人気の野球系TikTokerとのコラボ検討（低コスト・物々交換型）

#### X（高優先）
- 高校野球・中学野球関連のハッシュタグに参加
- 大会結果と連動した投稿
- 野球垢ユーザーとの積極的な交流（いいね・リプライ）

#### YouTube（中優先）
- サービス紹介動画（2-3分）を1本制作
- 使い方チュートリアル動画
- 「野球 成績 管理 アプリ」等のSEOキーワードを狙う

## 3. 口コミ・バイラル促進施策

### 3-1. チームメイト招待機能の活用

BUZZ BASEのグループ機能は口コミ獲得の最大の武器。1人が使い始めると「成績を比較する相手」が必要になるため、自然な招待動機が生まれる。

#### 施策

| 施策 | 内容 | 実装難易度 | 期待効果 |
| ---- | ---- | --------- | -------- |
| **招待リンク生成** | グループへの招待リンクをワンタップで生成・LINE/SNS共有 | 中 | 招待ハードルの大幅低下 |
| **「○人で始めよう」モーダル** | 初回登録後に「チームメイト3人を招待しよう」を表示 | 低 | 初期招待率の向上 |
| **招待実績バッジ** | 招待人数に応じたバッジをプロフィールに表示 | 低 | ゲーミフィケーションで招待動機付け |
| **チーム一括登録ガイド** | 監督・マネージャー向けに「チーム全員で始めるガイド」を用意 | 低 | チーム単位での一括導入 |

### 3-2. ランキング共有によるバイラル要素

#### 施策

| 施策 | 内容 | 実装難易度 | 期待効果 |
| ---- | ---- | --------- | -------- |
| **成績カード画像生成** | 個人成績をおしゃれなカード画像として生成→SNSシェア | 中 | Instagram/Xでの自然拡散 |
| **ランキングシェアボタン** | ランキング結果をワンタップでSNS共有 | 低 | 「俺1位だった！」の自慢投稿を促進 |
| **シーズン成績サマリー** | シーズン終了時に年間成績をまとめた画像を自動生成 | 中 | 年末年始の振り返り投稿で拡散 |
| **グループ対抗ランキング** | グループ同士で成績を比較できる機能 | 高 | チーム間の競争でバイラル効果 |

### 3-3. 紹介プログラム案

個人開発のため大規模なインセンティブは難しいが、ゲーミフィケーション要素で代替する。

| プログラム | 内容 | コスト |
| ---------- | ---- | ------ |
| **招待ランキング** | 「今月最も友達を招待したユーザー」をランキング表示 | 無料 |
| **アンバサダー制度** | 各チームから1名をBUZZ BASEアンバサダーに任命、限定バッジ付与 | 無料 |
| **フィードバック報酬** | 改善提案を採用したユーザーにプロフィールで特別表示 | 無料 |
| **早期登録特典**（アプリ版） | ネイティブアプリの事前登録者に限定テーマ・アイコンを付与 | 無料 |

## 5. ネイティブアプリリリース連携

### 5-1. リリース前プロモーション計画

| フェーズ | 時期 | 施策 | 目標 |
| -------- | ---- | ---- | ---- |
| **ティザー期** | リリース3ヶ月前 | SNSで開発進捗を公開、事前登録ページ開設 | 事前登録500件 |
| **カウントダウン期** | リリース1ヶ月前 | 毎週の機能プレビュー動画、既存Webユーザーへの通知 | 事前登録1,000件 |
| **ローンチ期** | リリース当日〜1週間 | SNS全チャネルで集中投稿、プレスリリース配信 | DL500件 |
| **ポストローンチ期** | リリース1ヶ月後 | ユーザーレビュー依頼、改善アップデート告知 | レビュー50件 |

### リリース日の選定

- **推奨**: 3月下旬〜4月上旬（新学期・新入部員シーズン）
- **次点**: 8月下旬〜9月上旬（新チーム始動シーズン）
- **避けるべき**: 12-2月（オフシーズンで野球への関心が低下）

### 5-2. ASO（App Store Optimization）基本施策

#### アプリ名・サブタイトル
```
BUZZ BASE - 野球の個人成績をランキングで共有
```

#### キーワード戦略

**主要キーワード（検索ボリューム重視）**
```
野球, 成績, 個人成績, スコア, 打率, ランキング, 記録, 野球部
```

**ロングテールキーワード（競合が少ない）**
```
野球成績管理, 個人成績記録, 野球ランキング, 打撃成績, 投手成績, 野球ノート, チーム成績比較
```

#### スクリーンショット戦略
1. ランキング画面（最大の差別化ポイント）
2. 成績入力画面（使いやすさ訴求）
3. グループ機能（ソーシャル要素）
4. 野球ノート（付加価値）
5. 成績グラフ・分析画面（データの価値）

#### その他のASO施策
- **アプリ説明文**: Google Playではキーワードを自然に含めた説明文（App Storeでは説明文はランキング要因ではないためブランドストーリー重視）
- **プレビュー動画**: 30秒以内で「成績入力→ランキング確認→友達と共有」の一連の流れを見せる
- **レビュー促進**: アプリ内で適切なタイミング（成績入力5回目など）にレビュー依頼ダイアログを表示
- **カテゴリ選択**: プライマリ「スポーツ」、セカンダリ「ソーシャルネットワーキング」
- **定期的な更新**: 2週間に1回のアップデートでストアでの評価を維持

| 項目 | 費用 | 備考 |

| SNS広告（テスト） | 5,000〜10,000円 | Instagram/TikTok広告の小規模テスト |
| デザインツール | 0円 | Canva無料プランで成績カード等を作成 |
