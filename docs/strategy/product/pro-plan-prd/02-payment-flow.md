## 概要

BUZZ BASE Pro のサブスクリプション課金フローの要件定義。
iOS / Web で同一の Pro 体験を、各プラットフォームのルールに従って提供する。

---

## 背景

- BUZZ BASE Pro はサブスクリプションモデル
- iOS では Apple IAP 必須、Web では Stripe を採用
- iOS と Web で別経路で課金されても、同じ Pro 機能を提供する必要がある
- Apple の App Store 規約に準拠する必要がある

---

## ゴール

- ユーザーが iOS / Web のどちらでも Pro 加入できる
- どちらのプラットフォームで加入しても、同じアカウントなら全プラットフォームで Pro 機能を利用できる
- 早期特典（30日無料）と通常トライアル（7日無料）を正しく出し分ける
- 解約・返金のフローが明確である
- 課金イベントが信頼性高く処理される（重複・遅延への耐性）

---

## Non-goals

- Android 課金（Google Play Billing）は本リリースでは対応しない（Phase 2 以降）
- プラン変更（月額↔年額）の初期実装（Phase 2 で対応）
- 複数ユーザーでの共有プラン・ファミリープラン（将来検討）

---

## 機能要件

### サブスクプラン

| プラン | 価格 | 通常トライアル | 早期特典トライアル |
|------|----|----------|----------|
| 月額プラン | ¥300 | 7日無料 | 30日無料 |
| 年額プラン | ¥2,980 | 7日無料 | 30日無料 |

### 早期特典の条件

- 期間: 2026-05-31 00:00:00 〜 2026-06-06 23:59:59 (JST)
- 対象: この期間内に Pro 申込を完了したユーザー
- 特典: 30日間の無料トライアル
- バッジ等の付与なし（プライバシー配慮）

### Apple 規約への準拠

- iOS アプリ内から Web 課金への誘導はしない
- iOS 内では iOS IAP のみで課金
- 「iOS ユーザーは iOS で、Web ユーザーは Web で」のクリアな設計

### 解約・課金失敗・返金・再加入の挙動

| ケース | 挙動 |
|----|----|
| 解約申請 | `cancelled` ステータスへ、期限まで Pro 機能利用可 |
| 期限到来 | `expired` ステータスへ、Pro 機能無効化、データは保持 |
| 課金失敗 | `billing_issue` ステータスへ、Grace Period 中は Pro 機能利用可 |
| Grace Period 終了 | `expired` ステータスへ、Pro 機能無効化 |
| 返金完了 | `expired` ステータスへ即時、Pro 機能即時無効化 |
| 再加入 | トライアルなし（1ユーザー1回）、過去データ即時復活 |

### 支払い方法

#### iOS

Apple ID に紐付いた支払い方法を自動使用（BUZZ BASE 側で選択させない）:
- クレジットカード
- Apple Pay
- キャリア決済（au, docomo, SoftBank）
- iTunes ギフトカード残高

#### Web（Stripe ダッシュボードで有効化）

- クレジットカード（Visa, Mastercard, JCB, AMEX）
- Apple Pay
- Google Pay
- Link（Stripe 保存決済）

## ユーザーストーリー

### US-01: 初めて Pro 加入する大学生（翔太）

> 翔太は5月31日のリリース当日に BUZZ BASE を開き、Pro 機能の案内モーダルを見る。
> 「30日無料」のメッセージに惹かれ、Pro 加入ボタンをタップ。
> Apple ID 認証だけで簡単に30日無料トライアルが始まる。

### US-02: Web で加入する社会人（健）

> 健は通勤中にスマホで BUZZ BASE の Web サイトを見る。
> Pro 加入バナーをタップ → Stripe Checkout 画面で決済。
> 決済完了後、即座に Pro 機能が解放される。
> 後で iOS アプリを開いても Pro 状態が同期されている。

### US-03: トライアル期間中に解約

> ユーザーがトライアル期間中に Pro が合わないと判断。
> 設定 → サブスクリプション管理 → 解約ボタンをタップ。
> トライアル期間中は一切課金されない。

### US-04: トライアル終了時の自動課金

> 7日（または30日）の無料期間終了時、自動的に月額¥300 / 年額¥2,980 が課金される。
> 課金開始3日前にプッシュ通知またはメールで予告される。

### US-05: 解約後の挙動

> 解約しても次の更新日まで Pro 機能を利用可能。
> 更新日に自動的に無料機能のみに切り替わる。

### US-06: 課金失敗（クレカ期限切れ等）

> クレカ期限切れで課金失敗した場合、Apple/Google/Stripe の Billing Grace Period 中は Pro を継続利用可能。
> ユーザーに通知され、決済情報の更新を促す。
> Grace Period 終了までに更新されないと、自動的に期限切れになる。

### US-07: 返金を受けたユーザー

> ユーザーが Apple/Stripe に返金請求し、承認される。
> 即座に Pro 機能が無効化される。
> 返金完了の通知をメールで受け取る。

### US-08: 解約後の再加入

> 一度解約したユーザーが再加入。
> トライアル期間はなく、即座に課金開始（1ユーザー1回まで）。
> 過去のデータがすべて復活し、即座に利用可能。

### US-09: 支払い方法の選択（Web）

> Web で加入する場合、Stripe Checkout でクレジットカード / Apple Pay / Google Pay / Link から支払い方法を選択できる。
> iOS で加入する場合、Apple ID 登録の支払い方法（クレカ、Apple Pay、キャリア決済等）が自動的に使われる。

---

## 早期特典 vs 通常トライアルの判定要件

- 申込日時 が 2026-05-31 00:00:00 〜 2026-06-06 23:59:59 (JST) の場合は30日無料
- それ以外の場合は7日無料
- 一人のユーザーは1回のみトライアル可能
- iOS / Web で同じユーザーが2回トライアルすることはできない

---

## クロスプラットフォーム同期要件

- Web で加入 → iOS にログインしたら Pro 状態が即座に反映される
- iOS で加入 → Web にログインしたら Pro 状態が即座に反映される
- 同期は RevenueCat を経由して行われる
- 同期遅延がある場合、ユーザーは「同期更新」を手動実行できる

---

## 解約フロー要件

### iOS

- Apple の規約により、アプリ内で解約処理はできない
- 設定 → Apple ID → サブスクリプション の画面へ誘導する
- アプリ内に「解約方法」を明示する

### Web

- 自前の解約画面で解約処理可能
- 解約確認画面で「次回課金日まで利用可能」を明示
- 解約完了画面で感謝メッセージ

---

## 課金イベントの信頼性要件

- Webhook イベントを重複処理しない（冪等性）
- Webhook 受信時に即座に2xx を返却する（10秒以内）
- イベントの順序が逆転しても正しい状態を維持する
- 一時的な障害時にもイベントを失わない（再処理可能）

詳細な実装方式は Design Doc を参照。

## UI 要件

### iOS - Pro 加入モーダル

- BUZZ BASE のタグライン「もっと野球がしたくなる」を冒頭に表示
- Pro 機能のスクリーンショット/アイコン表示
- 月額・年額の価格、トライアル期間を明示
- 「Pro に加入する」ボタン（メイン）
- 「後で」ボタン（サブ）

### Web - Pro 加入ページ（/pro）

- LP 用ヒーロー画像
- Pro 機能の詳細リスト
- 月額・年額の比較表
- リリース記念特典（30日無料）の明示
- 「ご安心ください - 既存機能は永久無料」を明示
- 「Pro に加入する」CTA ボタン

### 解約画面

- 現在のプラン、次回課金日、金額を表示
- 解約ボタン（注意喚起付き）
- 解約確認モーダル

---

## エッジケース対応要件

| ケース | 期待される挙動 |
|----|----|
| Apple ID 未連携で iOS IAP 失敗 | エラー表示、Apple ID 設定を促す |
| Web 決済中にネットワーク切断 | Stripe が自動リトライ、状態確認後通知 |
| トライアル期間中にプラン変更（月額→年額） | RevenueCat の PRODUCT_CHANGE で処理 |
| 二重課金（Web と iOS で別々に） | 2回目以降の購入をブロック |
| Webhook 受信失敗 | 自動リトライ、Sentry 通知 |
| 早期特典期間に多重申込 | 1ユーザー1回のみ、2回目は通常トライアル |
| 既に解約済みユーザーの再加入 | 通常の購入フロー（Apple のトライアル仕様に従う） |

## 完了の定義（Definition of Done）

- [ ] iOS Sandbox で IAP 経由の Pro 加入が動作
- [ ] Stripe Test Mode で Web 経由の Pro 加入が動作
- [ ] iOS / Web の Pro 状態が双方向に同期する
- [ ] 早期特典期間に申込んだユーザーが30日無料になる
- [ ] 通常期間に申込んだユーザーが7日無料になる
- [ ] トライアル中に解約しても課金されない
- [ ] トライアル終了時に自動課金が始まる
- [ ] 解約フローが iOS / Web で動作する
- [ ] Webhook の冪等性が機能する
- [ ] イベント順序非依存設計が機能する
- [ ] 監査ログに全イベントが記録される
- [ ] プライバシーマニフェスト（iOS 17+）対応済み
- [ ] 利用規約・プライバシーポリシー・特定商取引法表記が整備済み
- [ ] Sentry でエラー監視できる

## ファーストリリース追加要件（2026-05-17 追記）

レビューで確定した追加要件:

### 解約引き止め + 解約理由アンケート

- 解約ボタンタップ後に引き止めモーダル表示
- 「Pro 期間に蓄積したデータは閲覧不可になります」を明示
- 「使い続ける」ボタンと「解約を続ける」リンク
- 解約を選んだら理由アンケート（価格/利用頻度/機能不足/競合/その他）
- 業界知見: 20-30% が解約撤回

### UI: Pro 期限の表示

- 設定画面に Pro 状態と次回更新日を表示
- トライアル中: 「あと X日」
- cancelled: 「X月X日まで利用可」
- billing_issue: 警告バナー「決済情報を確認してください」

### プラン変更（月額↔年額）

- iOS: App Store の Subscription Group 内で自動対応
- Web: Stripe API で proration 計算
- 設定画面に「プラン変更」リンク

### アカウント削除前のサブスク強制解約

- Pro 加入中のユーザーがアカウント削除しようとした場合
- 「Pro 加入中のため、先に解約してください」を表示
- 解約完了後にアカウント削除可能

### RevenueCat Custom Attributes

- ユーザー属性（email、role、registration_date、is_early_subscriber）を RevenueCat に渡す
- ダッシュボードでセグメント分析可能

### グレース期間（Billing Grace Period）の設定

- Apple: App Store Connect で Billing Grace Period を **明示的に有効化（16日）**
- Stripe: Smart Retries がデフォルト有効、確認のみ

### 本番テスト戦略

- Web: Flipper で ippei のみ有効化 + Stripe Test Mode
- iOS: TestFlight 配信 + Sandbox テスター
