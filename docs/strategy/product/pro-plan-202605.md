## 進め方ガイド

ビジネスサイドの標準プロセス **WHY → WHO → WHAT → HOW MUCH → HOW** に沿って進める。
業界フレームワーク（Lean Startup / Working Backwards / INSPIRED / JTBD 等）と照合済み。

### 各フェーズで答える質問とフレームワーク

| フェーズ | 答える質問 | 使うフレームワーク |
|---------|---------|----------------|
| WHY | なぜユーザーは Pro にお金を払うのか? | JTBD（Jobs to be Done） |
| WHO | 誰が払うのか? 具体的に誰? | セグメンテーション、ペルソナ |
| WHAT | Pro は一言で何を提供するのか? | バリュープロポジション・キャンバス |
| WHAT' | 競合と比べてどう違うのか? | ポジショニングマップ、Five Forces |
| HOW MUCH | 成功とは何か? 数値で言うと? | KGI/KPI、NSM、AARRR、SaaS KPI |
| HOW MUCH' | 失敗とは何か? いつピボットするか? | Success/Failure Criteria、ピボット基準 |
| HOW（戦略） | どうやって獲得するか? いくらで? いくら稼げるか? | 4P、競合価格分析、収益試算 |
| HOW（実行） | 具体的に何を作るか? どう売るか? | MoSCoW、機能優先順位付け |
| Doc（顧客視点） | ローンチ時に何を伝えるか? | PR/FAQ（Amazon Working Backwards） |

### 順序の論理

- WHY/WHO/WHAT が決まらないと、KPI が「何を測るべきか」分からない
- HOW MUCH（KPI目標）が決まらないと、機能の優先順位を判断できない
- だから 1-3 で計測対象を明確化 → 4 で計測指標を設計 → 5-6 で逆算した戦略・機能
- PR/FAQ を PRD の前に書くことで「顧客視点で何を伝えるか」が明確化され、PRD の機能取捨選択が研ぎ澄まされる

### 2. ターゲットセグメント

**ステータス**: 🟢 決定（2026-05-12）

#### 決定内容

| 区分 | セグメント |
|------|---------|
| **メインターゲット** | 大学生・社会人（企業チーム・クラブチーム所属） |
| **サブターゲット** | 草野球（趣味層） |
| 将来的に取り込む層 | 中高生・シニア野球 |

#### 決定理由

1. **成長意欲の高さ**: 大学生・社会人（特に競技志向のチーム所属）は「**自分の成長にアプリを活用したい**」という動機が強く、サブスク継続率が高いと期待できる
2. **支払い能力**: 本人課金可能で可処分所得◎、Apple/Google税の壁を超えられる
3. **データ志向**: 検索クエリ分析から、現状ユーザーはセイバーメトリクス系指標を調べる層が多く、データ志向の大人層と一致
4. **草野球より優位な点**: 草野球は趣味の延長で成長意欲が低い可能性。Proの主要訴求「成長サポート」と整合しにくい
5. **既存サービスとの相性**: 現状のGAデータ（デスクトップ31%、セイバーメトリクス検索）と一致

#### サブターゲットとして草野球を残す理由

- 人口最大セグメント（60-80万人）で広告収益にも貢献
- 草野球層は「思い出記録」「仲間と共有」の文脈で利用するため、Pro機能の一部（推移グラフ、PDF出力）は刺さる可能性
- メインで取り込めなかった層をサブで救う設計

### 3.5. ユーザー課題（JTBD: Jobs to be Done）

**ステータス**: 🟢 決定（2026-05-12）

#### 決定内容: メインジョブ

> 野球を真剣にやっているとき、**「ライバルに負けたくない」「もっと上手くなりたい」「仲間と高め合いたい」という気持ち**を保ちたいので、**自分の取り組みを記録し、成長を実感し、仲間と繋がる**ことを通じて、**毎日のモチベーションを維持し、野球を諦めずに続ける**ために、BUZZ BASE Pro を雇用する。

#### 決定の核心: BUZZ BASE は感情ドリブン型サービス

世の中の個人成長サービスは2タイプに分類される:

| タイプ | 特徴 | 代表例 |
|--------|------|------|
| データツール型 | 数値の精度・分析の深さで勝負 | トラックマン、Garmin Connect |
| **感情ドリブン型** | データを「やる気を引き出す道具」として使う | **Strava、Duolingo、Peloton、Nike Run Club** |

**BUZZ BASE は感情ドリブン型として位置づける**。データは目的ではなく、モチベーションを引き出すための手段。

#### JTBD の3層分解

| 層 | ジョブ |
|---|------|
| **Emotional Job（中心）** | ライバルに負けたくない、もっと上手くなりたい、諦めたくない |
| **Social Job** | 仲間と高め合いたい、認められたい、応援されたい |
| Functional Job（手段） | 自分の取り組みを記録、成長を可視化、データで判断 |

→ 中心は感情・関係性。Functional は支える役割。

#### バリュープロポジションの方向性（ステップ4で確定）

候補:
- 「**野球を諦めない自分でいられる**」
- 「**毎日もう一度バットを振りたくなる**」

ユーザー承認済み: この方向で言葉を磨いていく。

#### 野球選手の成長要素マップ

BUZZ BASE が捕捉する要素を全方位で整理:

| 要素 | 比重 | データで捕捉 |
|------|------|------------|
| 練習量・取り組み方 | ★★★★★ | △ 練習ログ |
| **モチベーション・やる気** | **★★★★★** | ❌ 行動として表れる |
| 試合経験・実戦 | ★★★★ | ○ 既存記録 |
| **メンタル（負けたくない気持ち）** | **★★★★★** | ❌ 直接は計測不能 |
| 仲間・ライバル関係 | ★★★★ | ○ ランキング・SNS |
| 指導者からの学び | ★★★★ | △ メモ・ノート |
| 怪我・コンディション | ★★★ | △ 体調ログ |

データで扱える領域は半分以下。残り半分は **感情・関係性・継続性** で支える。

#### Pro機能の優先順位（ユーザー承認済み）

**感情・継続軸 ＞ 練習軸 ＞ データ可視化軸** の順で進める。

##### 🥇 感情・継続軸（最優先）
- 連続記録（Streak）
- 達成バッジ
- 過去の自分との比較（1年前の今日）

##### Q1: これまで使っていた機能は使えなくなりますか?

### データ取得の限界

- GA のデモグラフィック生データ（年齢層・性別）は **データのしきい値処理**（subject_to_thresholding: true）によりブロックされ、直接データは取得不可
- ユーザー数が一定以上に増えるか、Google Signals が有効化されると取得可能になる
- 代替案: アプリ内アンケート、SNSフォロワー分析、レビュー分析

### 結論

| 仮説 | 根拠 |
|------|------|
| 現状ユーザーは **大人寄り** の可能性が高い | デスクトップ31%、セイバーメトリクス検索多数 |
